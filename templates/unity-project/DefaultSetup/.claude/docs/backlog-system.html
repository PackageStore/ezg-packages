Unity Project · Backlog & Task Execution
Vòng đời một task: Capture → Queue → Execute. Bản chi tiết kỹ thuật: backlog-system.md

Cơ chế thực thi task tự động dựa trên backlog split-file: mỗi task là một file, BACKLOG.md chỉ là index — nên token phẳng dù backlog lớn cỡ nào. Bookkeeping deterministic (không hand-edit), spec là contract, không bịa quyết định rủi ro.

Ba lớp

CAPTURE

parallel-safe · không touch BACKLOG.md
  • /planning-task — cửa vào duy nhất, triage + spec
  • /planning-system — GDD lớn → N task
  • /ui-mockup — ground truth UI (human approve)
→ backlog/planning/*.md

QUEUE

serial · intentional commit của user
  • /add-to-backlog
  • promote --check → 3 hard blocker
  • promote → gán NNN, update index
→ backlog/todo/*.md + BACKLOG.md

EXECUTE

autonomous · quality gates + auto-fix
  • /run-backlog
  • pick → branch → implement → gates
  • DONE → commit + push agent/dev
→ in-progress → done/ (no PR)

/planning-system — GDD lớn → N task

Design-first: validate thiết kế qua các stage, rồi batch-ground mapping thành N task theo dependency order. Profile co giãn độ sâu.

LITE

Không economy/backend VÀ ≤5 sub-feature → bỏ 03, 05 chỉ Lens 1+3. Cắt ~40–60% token.

STANDARD

Economy HOẶC 6–15 sub-feature. Mặc định khi thiếu tín hiệu. Full flow.

EPIC

Backend/PvP/social HOẶC >15 sub-feature → Module Split → 05+06 per-module.

Stage pipeline (subagent/stage, stage sau ăn artifact stage trước):

04 feature-analysis → Architecture.md · 05 tech-spec → TechSpec.md (4-lens audit) · 06 mapping → Implementation.md (§10.1–10.7)

stage-result {OK | QUESTIONS | ABORT}: QUESTIONS → hỏi user ≤3 câu → re-spawn ĐÚNG stage. Batch output: MỘT timestamp + NN topo (= thứ tự thực thi), mỗi task mang Context docs + Depends on; trộn nhiều tier.

Mockup pipeline — UI ground truth (tích hợp ui-catalog)

Capability gate: UI-kit sinh từ ui-catalog của project hiện tại; ui-kit-sync.py extract geometry/màu từ prefab thật. Fresh project chưa export catalog phải dừng với prerequisite rõ ràng, không copy generated catalog từ game khác. Template name trong spec = token id (ui.currency.single). Default contract 1080×2400.

Generate = autonomous (parallel-safe) · Approve = HUMAN-ONLY (serial, bất đồng bộ — không dừng planning):

groundTruthNghĩa
...<S>.pngĐã approve — PNG đóng băng là contract
PENDING-APPROVAL:<html>Draft có, chờ human duyệt ở /ui-mockup
PENDING-MOCKUPChưa draft — /ui-mockup sẽ draft nốt

/add-to-backlog — pick vào queue

1–3
LIST → DISPLAY → PICK — theo task order (timestamp, NN); user chọn 1 / 1,3 / 1-3 / all
4.5
PRECHECK — promote --check: 3 hard blocker → tier_errors dependency_warnings mockup_warnings
5
PROMOTE — gán NNN + git mv planning→todo + append bullet + self-lint. Sort theo task order, KHÔNG bucket priority (giữ dependency chain).

/run-backlog — pipeline 11 bước

Loop tự động: pick task đầu, implement, qua quality gate (auto-fix ≤2 vòng/gate), mark done, push agent/dev. Không tạo PR.

1
PICK — backlog-ops pick (todo | in-progress resume | empty → PAUSED)
1b
REQUIRES — **Requires:** unity-editor + Editor không sống → defer (còn task headless) hoặc EDITOR_REQUIRED
2
BRANCH — checkout agent/dev + merge $BASE_BRANCH (conflict → BRANCH_BLOCKED)
3
START — backlog-ops start: todo → in-progress
4
CONTEXT — đọc rules + task + $CONTEXT_DOCS + code liên quan
5
IMPLEMENT — 5.0 WF shortcut (load command, Context docs input ưu tiên 0, /new-ui groundTruth gate) → code → git add → 5b 3-tier compile → 5c tier guard
6
REVIEW — preflight → detect sensitive/perf → spawn code-reviewer (+perf +security) song song → auto-fix ≤2
7
VERIFY — qa-verifier → auto-fix ≤2 → final preflight
7.5
SMOKE — runtime smoke (M/L, orchestrator chạy Unity MCP): play mode + console assert + invariant suite + screenshot
8–9
DONE + SHIP — backlog-ops done + completion summary → lint → commit + push agent/dev

Tier guard — cắt gate theo tier (STEP 5c)

TierGate chạy
XSPreflight. Pass → thẳng DONE. Bỏ mọi reviewer + verify + smoke.
SPreflight + code-reviewer (+perf nếu perf-sensitive +security nếu sensitive). Bỏ qa-verifier + smoke.
M/LFull pipeline: STEP 6 + 7 + 7.5.

Compile check 3-tier (5b)

1 Unity Editor MCP → 2 dotnet build → 3 Unity batch mode. Dừng ở tier đầu chạy được; fix loop ≤2 vòng; skip chỉ khi cả 3 không chạy được. (Unity không có npm run lint.)

backlog-ops.py & chạy loop

backlog-ops.py  lint | pick | start <NNN> | done <NNN> | demote <NNN>
                defer <NNN>                    # task cần Editor → cuối queue
                promote <planning.md>...        # gán NNN-TIER, block khi đứt dep/mockup
                promote --check <f>...          # preflight read-only
                timestamp                       # filename planning

# macOS loop runner (controller, mỗi task 1 cửa sổ Terminal):
.claude/scripts/run-backlog-loop.sh --auto-model-by-tier --max-iterations 5
